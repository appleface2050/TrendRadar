# 智能知识库上传脚本使用指南

## 概述

智能上传脚本（smart-upload.py）解决了重复上传的问题，通过文件哈希值检测文件变化，只上传新文件或已更改的文件。

### 核心优势

- ✅ **避免重复上传**：自动跳过未更改的文件
- ✅ **节省时间**：只上传需要更新的文件
- ✅ **可追溯性**：维护上传记录文件，跟踪文件版本
- ✅ **可选删除旧版本**：支持删除知识库中的旧版本

---

## 工作原理

### 文件哈希检测

脚本使用 MD5 哈希算法计算每个文件的唯一标识：

```python
def get_file_hash(file_path):
    hash_md5 = hashlib.md5()
    with open(file_path, "rb") as f:
        for chunk in iter(lambda: f.read(4096), b""):
            hash_md5.update(chunk)
    return hash_md5.hexdigest()
```

### 上传记录

上传记录保存在 `/home/shang/git/Indeptrader/knowledge-base/.upload_record.json`，包含以下信息：

```json
{
    "文件名.md": {
        "hash": "文件内容的 MD5 哈希值",
        "document_id": "BigModel.cn 知识库中的文档 ID",
        "upload_time": "上传时间（ISO 8601 格式）",
        "file_size": 文件大小（字节）,
        "file_mtime": 文件修改时间（Unix 时间戳）
    }
}
```

### 上传逻辑

```
┌─────────────────┐
│  扫描目录文件    │
└────────┬────────┘
         │
         ▼
┌─────────────────┐    是    ┌─────────────────┐
│ 文件已上传过？   │──────────►│ 哈希值未变化？   │
└────────┬────────┘          └────────┬────────┘
         │                            │
         │ 否                         │ 是
         ▼                            ▼
┌─────────────────┐            ┌─────────────────┐
│    上传文件      │            │    跳过文件      │
└─────────────────┘            └─────────────────┘
```

## 使用方法

### 方式一：快速上传（推荐）

```bash
./scripts/smart-upload.py
```

这个脚本会：
1. 自动使用统一知识库目录 `/home/shang/git/Indeptrader/knowledge-base`
2. 调用智能上传脚本
3. 显示详细的上传统计

### 方式二：直接使用智能上传脚本

```bash
python3 /home/shang/git/Indeptrader/knowledge-base/scripts/smart-upload.py \
  --dir /home/shang/git/Indeptrader/knowledge-base
```

### 方式三：上传并删除旧版本

```bash
python3 /home/shang/git/Indeptrader/knowledge-base/scripts/smart-upload.py \
  --dir /home/shang/git/Indeptrader/knowledge-base \
  --delete-old
```

使用 `--delete-old` 参数时，如果文件已更改，脚本会自动删除知识库中的旧版本。

## 故障排查

### 问题 1：所有文件都被重新上传

**症状**：运行脚本时，之前上传过的文件又被重新上传。

**可能原因**：
1. 上传记录文件被删除或损坏
2. 文件修改时间发生变化，但内容未变化

**解决方案**：
```bash
# 检查上传记录文件是否存在
cat /home/shang/git/Indeptrader/knowledge-base/.upload_record.json

# 如果文件不存在或损坏，删除后重新运行（会重新上传所有文件）
rm /home/shang/git/Indeptrader/knowledge-base/.upload_record.json
./scripts/smart-upload.py
```

### 问题 2：文件被跳过但应该上传

**症状**：文件已更改，但脚本显示"跳过（未更改）"。

**可能原因**：
1. 文件内容实际未变化
2. 文件哈希值计算错误

**解决方案**：
```bash
# 手动验证文件哈希
python3 -c "
import hashlib
hash_md5 = hashlib.md5()
with open('/path/to/file.md', 'rb') as f:
    for chunk in iter(lambda: f.read(4096), b''):
        hash_md5.update(chunk)
print(hash_md5.hexdigest())
"

# 对比上传记录中的哈希值
cat /home/shang/git/Indeptrader/knowledge-base/.upload_record.json | grep 文件名
```

### 问题 3：符号链接文件被跳过

**症状**：使用符号链接的文件无法正常上传。

**解决方案**：
- 脚本会自动解析符号链接
- 如果符号链接损坏，重新运行 `setup-knowledge-base.sh`

```bash
/home/shang/git/Indeptrader/knowledge-base/scripts/setup-knowledge-base.sh
```
