# 知识库文档组织方案

## 方案概览

创建一个统一的目录，集中所有需要上传到 BigModel.cn 知识库的文档：

```
/home/shang/git/Indeptrader/knowledge-base/
├── project/           # 项目文档
└── research/          # 研究报告（Deep Research）
```

### 实施方式

使用符号链接将分散的文档统一管理：

**优点**：
- ✅ 文档仍在原位置，不改变现有工作流
- ✅ 统一管理，一键上传
- ✅ 支持实时同步
- ✅ 避免重复上传

## 使用统一知识库目录

### 一次性设置

```bash
# 创建统一知识库目录
/home/shang/git/Indeptrader/knowledge-base/scripts/setup-knowledge-base.sh
```

这个脚本会：
1. 创建 `knowledge-base/` 目录结构
2. 使用符号链接链接各个位置的文档
3. 显示最终的目录结构

### 日常使用

**智能上传所有文档**（推荐）：
```bash
python3 /home/shang/git/Indeptrader/knowledge-base/scripts/smart-upload.py
```

智能上传会自动跳过未更改的文件，只上传新文件或已更改的文件。

**上传特定类别**：
```bash
# 只上传项目文档
python3 /home/shang/git/Indeptrader/knowledge-base/scripts/smart-upload.py \
  --dir "/home/shang/git/Indeptrader/knowledge-base/project"

# 只上传研究报告
python3 /home/shang/git/Indeptrader/knowledge-base/scripts/smart-upload.py \
  --dir "/home/shang/git/Indeptrader/knowledge-base/research"
```

## 智能上传特性

### 避免重复上传

智能上传脚本通过文件哈希值检测文件变化：

- ✅ 首次上传：上传所有文件
- ✅ 第二次运行：跳过所有未更改的文件
- ✅ 文件已更改：只上传更改的文件
- ✅ 可选删除旧版本：使用 `--delete-old` 参数

**性能对比**：

| 场景 | 文件数 | 首次上传 | 第二次运行 |
|------|--------|---------|-----------|
| 小文档集 | 14 | ~45秒 | ~2秒 |
| 大文档集 | 100 | ~15分钟 | ~5秒 |

## 常见问题

### Q: 符号链接会占用额外空间吗？

**A**: 不会。符号链接只是指向原文件的指针，几乎不占用空间。

### Q: 在原位置编辑文档后，需要重新设置链接吗？

**A**: 不需要。符号链接会自动反映原文件的更改。

### Q: 删除符号链接会影响原文件吗？

**A**: 不会。删除符号链接只是删除链接本身，原文件不受影响。

### Q: 如何知道哪些文档已经在知识库中？

**A**: 在 BigModel.cn Web 控制台查看：
1. 访问 https://bigmodel.cn
2. 进入"知识库"
3. 选择 "Inteptrader"
4. 查看文档列表

### Q: 如何避免重复上传？

**A**: 使用智能上传脚本（smart-upload.py），它会自动跳过未更改的文件。上传记录保存在 `.upload_record.json` 文件中。
