# 优化数据清洗 Pipeline

## 变更 ID
`optimize-data-cleaning-pipeline`

## 类型
优化 / 重构

## 动机

### 当前问题
1. **代码重复**：`convert_existing_data.py` 中的多个函数存在大量重复逻辑
   - 日期格式转换代码重复
   - CSV 读写代码重复
   - 错误处理模式重复

2. **缺乏统一的数据验证**：
   - 没有统一的数据质量检查（缺失值、异常值、重复值）
   - 缺少数据格式验证
   - 没有数据完整性检查

3. **错误处理不一致**：
   - 有些函数有 try-except，有些没有
   - 错误信息不统一，难以调试

4. **缺少日志记录**：
   - 只有 print 语句，没有结构化日志
   - 无法追踪数据处理历史

5. **缺少单元测试**：
   - 数据处理逻辑没有测试覆盖
   - 重构时容易引入 bug

### 影响
- 维护成本高：修改数据处理逻辑需要在多处修改
- 调试困难：数据质量问题难以追溯
- 扩展性差：添加新数据源需要重复编写相似代码

## 目标

### 主要目标
1. 提取通用的数据处理组件，减少代码重复
2. 建立统一的数据验证和质量检查机制
3. 标准化错误处理和日志记录
4. 添加单元测试覆盖核心逻辑

### 次要目标
1. 提高代码可读性和可维护性
2. 为未来添加新数据源提供更好的扩展性

### 非目标
1. 不改变现有数据文件的结构和格式
2. 不修改数据获取逻辑（API 调用部分）
3. 不影响现有的回测流程

## 提议的解决方案

### 1. 创建数据处理工具库
```
MacroTrading/utils/data_processing.py
```

**核心组件**：
- `DataProcessor` 基类：提供通用数据处理方法
- `DataValidator` 类：数据验证和质量检查
- `DataLogger` 类：统一的日志记录

### 2. 重构现有转换脚本
将 `convert_existing_data.py` 中的函数改写为使用新的工具库

### 3. 添加数据质量报告
生成数据处理报告，包括：
- 处理记录数
- 缺失值统计
- 异常值检测
- 时间范围验证

### 4. 添加单元测试
```
MacroTrading/tests/test_data_processing.py
```

## 成功标准

- ✅ 所有现有的 `convert_*` 函数保持相同的输入输出接口
- ✅ 数据处理结果与重构前完全一致
- ✅ 添加数据质量检查，能自动检测常见问题

### 质量性
- ✅ 代码重复率降低 50% 以上
- ✅ 核心数据处理逻辑有单元测试覆盖
- ✅ 所有函数有统一的错误处理和日志记录

### 性能
- ✅ 数据处理速度不低于重构前
- ✅ 内存使用不显著增加

## 风险和缓解措施

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 重构引入 bug | 数据处理结果错误 | 1. 添加单元测试<br>2. 对比重构前后的输出<br>3. 分阶段发布 |
| 接口变化导致其他模块出错 | 回测失败 | 保持现有函数接口不变，仅重构内部实现 |
| 性能下降 | 数据处理变慢 | 使用性能基准测试，确保不低于重构前 |

## 时间估算
- 创建数据处理工具库：2-3 小时
- 重构现有脚本：1-2 小时
- 添加单元测试：1-2 小时
- 测试和验证：1 小时

**总计**：5-8 小时

## 替代方案

### 方案 A：最小化重构
只提取最常用的函数，不做大规模重构
- **优点**：风险低，工作量小（2-3 小时）
- **缺点**：代码重复仍然存在

### 方案 B：完全重新设计
设计全新的数据处理框架
- **优点**：架构最优
- **缺点**：工作量大（15-20 小时），风险高

**选择方案**：当前的平衡方案（方案 C）
